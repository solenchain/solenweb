Rules

  • Projects must deploy to and interact with the Solen testnet
  • Teams of 1-4 people. Solo hackers welcome.
  • Code must be open source (public GitHub repo)
  • Submit a working demo — video, live link, or detailed README with screenshots
  • Projects started before the hackathon are allowed, but only work done during the event is judged
  • Use any language for your frontend/tooling, but on-chain contracts must be Rust compiled to WASM
  • Be respectful. No plagiarism, no spam submissions.
